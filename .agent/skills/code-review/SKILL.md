# Code Review Skill

在完成功能开发或修复后，提交代码前使用此技能进行自我审查。

## 目标
确保代码质量，发现潜在 Bug，统一代码风格，并确认需求是否完全实现。

## 流程指令

请对照以下清单（Checklist）检查代码：

1.  **功能正确性 (Correctness)**
    *   [ ] 代码是否完全实现了 `task.md` 或设计文档中的需求？
    *   [ ] 边界情况（Edge cases）是否已处理？（如空值、极大值、网络异常）
    *   [ ] 是否通过了所有相关测试？

2.  **代码质量 (Quality)**
    *   [ ] 命名是否清晰、准确？（变量、函数、类名）
    *   [ ] 是否存在重复代码？(DRY)
    *   [ ] 函数是否过长或过于复杂？(SRP)
    *   [ ] 是否有必要的注释？（解释“为什么”而不是“是什么”）

3.  **安全性 (Security)**
    *   [ ] 是否存在注入风险？（如 SQL 注入、XSS）
    *   [ ] 敏感信息是否已脱敏或加密？
    *   [ ] 权限校验是否完善？

4.  **性能 (Performance)**
    *   [ ] 是否有明显的性能瓶颈？（如循环内查询数据库）
    *   [ ] 资源是否正确释放？

## 在发现问题时
*   按以下等级分类问题：
    *   **🔴 Critical**: 必须立即修复的错误（Bug、安全漏洞）。
    *   **🟡 Major**: 强烈建议修复的问题（设计缺陷、性能隐患）。
    *   **🟢 Minor**: 可以暂缓的建议（命名风格、注释）。
*   生成一份审查报告，并询问用户是否需要立即修复这些问题。

## 注意事项
*   审查时要像一个挑剔的高级工程师。
*   全程使用**简体中文**。
