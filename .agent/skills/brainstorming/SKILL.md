# Brainstorming Skill

当用户有新想法、模糊需求或需要讨论设计方案时，请使用此技能。

## 目标
1.  **明确目标**：理解用户真正想要解决的问题。
2.  **探索方案**：分析至少两种技术实现路径。
3.  **达成共识**：生成一份用户认可的设计文档或 PRD。

## 流程指令
请严格按照以下步骤与用户互动：

1.  **需求澄清 (Clarification)**
    *   不要急于给出解决方案。
    *   使用苏格拉底式提问来挖掘隐藏的需求（例如：“这个功能的主要目标用户是谁？”“在极端情况下系统应该如何表现？”）。
    *   确认非功能性需求（性能、安全性、兼容性等）。

2.  **方案分析 (Analysis)**
    *   提出至少两个可行的方案。
    *   对比各方案的优缺点（Pros/Cons）。
    *   推荐通过“奥卡姆剃刀”原则筛选出的最简方案。

3.  **文档生成 (Documentation)**
    *   当方案确定后，创建一个 Markdown 格式的设计文档（如果已有 PRD 则更新它）。
    *   文档应包含：
        *   问题背景
        *   目标与非目标
        *   详细设计（API 接口、数据结构、流程图等）
        *   验证计划

## 示例互动
User: "我想加个购物车功能。"
Agent: "好的。在开始之前，我想明确几个细节：
1. 购物车的数据是存在本地还是服务器？
2. 是否需要支持未登录用户的购物车？
3. 商品数量变更时是否需要实时计算促销价格？"

## 注意事项
*   全程使用**简体中文**。
*   保持客观，敢于指出用户需求中的逻辑漏洞。
