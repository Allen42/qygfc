# Writing Plans Skill

当设计方案已确定，准备进入编码实施阶段时，使用此技能。

## 目标
将高层设计转化为 `task.md` 文件中的具体、可执行的操作步骤。

## 流程指令

1.  **读取设计**
    *   回顾已确认的 PRD 或设计文档。
    *   理解核心功能点和依赖关系。

2.  **任务拆解 (Task Breakdown)**
    *   将每个功能点拆解为独立的开发任务。
    *   **粒度控制**：每个任务的预计耗时应在 2-10 分钟内。
    *   如果任务过大，继续拆分。

3.  **制定验证标准 (Verification Criteria)**
    *   为每个任务定义明确的“完成标准”（Definition of Done）。
    *   指明如何验证该任务（例如：运行特定测试命令、检查特定 UI 元素）。

4.  **生成/更新 Task 文件**
    *   在 `task.md` 中以 Markdown 复选框列表格式输出。
    *   格式示例：
        ```markdown
        - [ ] 任务名称
            - 目标：...
            - 文件：`path/to/file`
            - 验证：运行 `npm test xyz`
        ```
        
5.  **将执行计划转为需求文档以markdown格式保存到项目的PRD目录中，如果没有PRD目录则创建PRD目录。**        

## 注意事项
*   任务顺序应符合依赖逻辑（先底层后上层）。
*   必须包含**创建测试**或**验证逻辑**的任务。
*   全程使用**简体中文**。
